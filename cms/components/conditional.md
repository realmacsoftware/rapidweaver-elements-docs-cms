# Conditional

The **Conditional** component allows you to control when content is displayed based on specific rules or property values within your CMS collections. It acts as a filter or gatekeeper, showing or hiding parts of a page depending on the data available.

Each condition checks a **Property**, compares it with a **Value** using an **Operator**, and determines whether the condition passes. You can create multiple conditions and choose whether _all_ or _any_ of them must be met before the content inside the component is rendered.

The Conditional component helps you build smarter, more dynamic CMS-driven sites — without needing custom code or templates.
